### 4. AI 착장샷 (Virtual Fitting)

**Figma Frame**: (링크)

> **핵심 페이지**: LookFit의 차별화 기능

**사용자 플로우**:
1.  **사진 업로드**:
    -   사용자는 자신의 전신 사진을 업로드합니다.
    -   가이드라인에 따라 적절한 포즈를 취하도록 안내합니다.
2.  **옷 선택**:
    -   사진 업로드 후, 또는 제품 상세 페이지에서 진입 시 선택된 옷을 확인합니다.
    -   (향후 확장 시) 여러 옷을 선택/제거하는 기능 추가.
3.  **AI 처리 중**:
    -   업로드된 사진과 선택된 옷으로 AI 착장샷 생성을 요청합니다.
    -   AI 처리 시간(약 30초) 동안 대기합니다.
4.  **결과 확인**:
    -   생성된 AI 착장샷을 확인합니다.
    -   원본 사진과 착장샷을 비교하여 볼 수 있습니다.
5.  **저장/공유/구매**:
    -   만족스러운 결과는 저장, 공유하거나 장바구니에 추가하여 구매할 수 있습니다.

**레이아웃 구조**:

**[사진 업로드 단계]**
```
┌─────────────────────────────────┐
│ Header                          │
├─────────────────────────────────┤
│ Upload Area                     │  ← 드래그앤드롭 + 파일 선택 버튼
│   - Text: "사진을 업로드해주세요" │
│   - Subtext: "JPG, PNG, WebP 파일 (최소 512x768px)" │
│   - Icon: camera                │
├─────────────────────────────────┤
│ Pose Guide                      │  ← 적절한 포즈 가이드라인 이미지/텍스트
├─────────────────────────────────┤
│ Footer                          │
└─────────────────────────────────┘
```

**[AI 처리 중 단계]**
```
┌─────────────────────────────────┐
│ Header                          │
├─────────────────────────────────┤
│ Loading Animation               │  ← 스피너, 프로그레스 바 등
│   - Text: "AI 착장샷을 생성 중입니다..." │
│   - Subtext: "약 30초 정도 소요됩니다." │
├─────────────────────────────────┤
│ Footer                          │
└─────────────────────────────────┘
```

**[결과 화면 단계]**
```
┌─────────────────────────────────┐
│ Header                          │
├─────────────────────────────────┤
│ Result Comparison               │  ← 원본 사진 | AI 착장샷 (Desktop)
│                                 │  ← 원본 사진 위에 AI 착장샷 (Mobile - 탭/슬라이더)
├─────────────────────────────────┤
│ Action Buttons                  │
│   - "다른 옷 시도" (Secondary Button) │
│   - "장바구니 추가" (Primary Button) │
│   - "공유" (Ghost Button with share icon) │
├─────────────────────────────────┤
│ Footer                          │
└─────────────────────────────────┘
```

**반응형 브레이크포인트**:
-   **Desktop (1024px 이상)**:
    -   사진 업로드/결과 화면: 2컬럼 레이아웃. 업로드 영역 또는 비교 화면이 주를 이룸.
    -   결과 비교: 원본 사진과 AI 착장샷을 나란히 (Side-by-side) 표시.
-   **Mobile (1023px 이하)**:
    -   모든 섹션 1컬럼 레이아웃, 풀스크린 차지.
    -   터치 영역을 고려하여 버튼 및 요소 크기 확대.
    -   결과 비교: 탭 또는 슬라이더를 통해 원본과 착장샷을 전환하여 볼 수 있도록. (Ex: 좌우 스와이프 또는 탭 선택)

**업로드 UI**:
-   **드래그앤드롭 영역**: `var(--color-neutral-100)` 배경, `var(--color-neutral-300)` 점선 테두리, `var(--radius-lg)`. 높이 `200px` (모바일), `300px` (데스크톱).
-   **파일 선택 버튼**: `Primary` 버튼 스타일, "사진 업로드" 텍스트. 카메라 아이콘 (`camera`).
-   **가이드라인 (자세 안내)**: 텍스트는 `var(--color-text-secondary)`. 적절한 포즈 예시 이미지 포함. 폰트 크기 `var(--text-sm)`.

**로딩 상태**:
-   **프로그레스 표시**: `Skeleton` 컴포넌트 활용 또는 사용자 정의 `ProgressBar` (Primary Color).
-   **예상 시간**: "약 30초 소요" 텍스트 (`var(--color-text-tertiary)`).
-   **애니메이션**: 스피너(`var(--color-primary-500)`) 및 진행률 텍스트 `var(--text-lg)`.

**결과 화면**:
-   **원본 vs 착장샷 비교**:
    -   이미지 크기: 데스크톱에서는 3:4 비율로 두 이미지를 나란히. 모바일에서는 화면 너비에 맞게 조정.
    -   WebP 포맷, 768x1024px (`shared-conventions.md` 참조).
-   **다른 옷 시도 버튼**: `Secondary` 버튼 스타일.
-   **장바구니 추가 버튼**: `Primary` 버튼 스타일, 장바구니 아이콘 (`cart`) 포함.
-   **공유 버튼**: `Ghost` 버튼 스타일, 공유 아이콘 (`share`) 포함.

#### FileUpload

**구조**:
```
┌─────────────────────────────────┐
│               Icon              │  ← camera (32x32px)
│               Text              │  ← "사진을 업로드해주세요"
│              Subtext            │  ← "JPG, PNG, WebP 파일 (최소 512x768px)"
│       [파일 선택 버튼]          │  ← Primary Button
└─────────────────────────────────┘
```

**치수**:
-   전체 영역: 높이 `200px` (모바일), `300px` (데스크톱). 너비 `100%`.
-   패딩: `var(--space-6)` 상하좌우.
-   버튼: `md` 사이즈.

**색상**:
-   배경: `var(--color-neutral-50)` (호버 시 `var(--color-neutral-100)`)
-   테두리: `var(--color-neutral-200)` (점선)
-   아이콘/텍스트: `var(--color-text-tertiary)`

**상태**:
-   Default: 위 치수 및 색상.
-   Hover: 배경 `var(--color-neutral-100)`.
-   Drag Over: 테두리 `var(--color-primary-500)`.

---

#### ProgressBar

**구조**:
```
┌─────────────────────────────────┐
│         [진행률 텍스트]         │  ← ex) 60%
│ ┌───────────────────────────┐   │
│ │ █ █ █ █ █ █ █ █ █ █ █ █ █ │   │  ← 진행 바
│ └───────────────────────────┘   │
└─────────────────────────────────┘
```

**치수**:
-   높이: `8px`.
-   진행 바 너비: `100%`.
-   텍스트: `var(--text-lg)`.

**색상**:
-   배경 (바): `var(--color-neutral-200)`
-   진행 (바): `var(--color-primary-500)`
-   텍스트: `var(--color-text-primary)`

**애니메이션**:
-   진행률 변화 시 부드러운 트랜지션 (`--duration-normal`, `--ease-in-out`).

## 작성 가이드 (Gemini용)

### 이 문서 작성 시 포함해야 할 것

1. **Figma 링크**: 각 페이지/컴포넌트의 Figma Frame 링크

4. **색상**: `shared-conventions.md`의 토큰명 사용
5. **타이포그래피**: 폰트 크기, 굵기, 행간

```

치수:
```
